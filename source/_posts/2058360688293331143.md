这篇 X 长文系统介绍了 Agent Skills：它把某类任务的执行方法沉淀成可复用的能力模块，让 Claude Code、Codex、Cursor、Gemini CLI 等工具能够按同一套 SOP 稳定完成工作。

原文入口：<https://x.com/ai_xiaomu/status/2058360688293331143>

## 主要观点

1. Skill 不是 Claude 专属能力，而是一种可迁移的 Agent Skills 组织方式。
2. Skill 的核心通常是一个 `SKILL.md` 文件，用来描述触发场景、执行流程、约束、输出格式和可复用资源。
3. 它和普通提示词的区别在于可复用、可版本化、可自动触发，更适合长期工作流沉淀。
4. 它和知识库的区别在于 Skill 更偏“怎么做”，知识库更偏“知道什么”。
5. 它和 MCP 的关系更像流程与工具：Skill 告诉 Agent 如何完成任务，MCP 提供可调用的外部能力。
6. 好的 Skill 不应该只堆一段超长提示词，而应该拆成清晰的目标、步骤、边界和示例。

## 适合谁看

- 想把重复提示词沉淀成固定流程的人；
- 经常用 Claude Code、Codex、Cursor 或 Gemini CLI 做自动化任务的人；
- 想为写作、运营、开发、研究、数据分析建立可复用 SOP 的人；
- 想区分 prompt、RAG、MCP、Agent、Skill 这些概念的人。

## 我的笔记

Skill 最值得关注的点，不是“又多一种提示词写法”，而是把个人经验从一次性对话里抽出来，变成 Agent 可以长期继承的工作习惯。

如果一个任务满足这些条件，就很适合做成 Skill：

- 经常重复；
- 有固定输入和输出；
- 有明确质量标准；
- 需要多个步骤；
- 你不想每次都重新解释背景。

对博客、内容运营和开发工作流来说，Skill 可以成为很好的“个人操作系统插件”。比如：X 帖同步成博客、公众号排版、代码审查、封面设计、选题筛选，都可以各自沉淀为独立 Skill。

## 延伸方向

后续可以把博客同步流程也整理成一个 Skill：

- 输入 X/Twitter URL；
- 使用 Jina Reader 获取 Markdown；
- 判断是普通帖子还是长文；
- 生成 Hexo front matter；
- 保存到 `source/_posts/`；
- 运行构建验证；
- 提交并推送 GitHub；
- 由 Cloudflare Pages 自动部署。
